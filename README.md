# ★ 백워드 AI (Backward AI) - 초등 백워드 설계·과정 중심 평가 올인원 플랫폼

교사가 수업 소재 한 줄만 입력하면, 2022 개정 교육과정 초등 핵심 성취기준 611개에서 최적의 교과 융합 조합을 찾아내고, 평가 루브릭 → 결재 문서 표준 양식의 수업 과정안 → 수업별 코드(QR) 기반 학생 평가 수합 → AI 평가·피드백 → 학교생활기록부 문구 초안까지, 백워드 설계(Backward Design)의 전 과정을 하나의 흐름으로 완성하는 에듀테크 플랫폼 '백워드 AI'를 제출합니다. 본 서비스는 과정 중심 평가의 이상과 교육 현장의 현실 사이의 고민에서 탄생했습니다.

---

## 1. 기획 의도

'평가를 먼저 설계하고 수업을 거꾸로 계획하라'는 백워드 설계는 과정 중심 평가의 정석이지만, 교실 현장에서 실천하기에는 장벽이 높습니다. 611개 성취기준 문서를 뒤지며 융합 조합을 찾고, 루브릭 표를 그리고, 결재용 과정안을 쓰고, 학생 평가지를 만들어 걷고, 채점하고 피드백을 쓰고, 생기부 문구까지 다듬는 일이 각각 따로 노는 문서 작업으로 쪼개져 있기 때문입니다. 이에 "성취기준 선택이라는 첫 단추만 끼우면, 평가와 수업과 기록이 하나의 데이터 흐름으로 이어지는 조력자가 있다면 어떨까?"라는 문제의식에서 출발했습니다. 단순히 문서를 생성해 주는 것을 넘어, 앞 단계의 확정 결과가 다음 단계의 입력이 되는 백워드 설계의 논리 구조 자체를 소프트웨어로 구현하여 평가-수업-기록의 일관성을 시스템이 보장하도록 하였습니다.

---

## 2. 핵심 기능

### 1) 2단계 AI 융합 성취기준 추천 (창의적 소재 완전 대응)

'강낭콩 키우기'는 물론 '선관위', '월드컵'처럼 교육과정 문서에 없는 창의적 소재도 추천이 가능합니다. AI가 먼저 소재를 교과 개념 키워드(선관위 → 선거·민주주의·투표·시민)로 번역한 뒤, 611개 성취기준을 재검색하여 과목을 넘나드는 융합 조합과 추천 사유를 제시합니다. AI가 존재하지 않는 성취기준 코드를 지어내지 못하도록 후보 화이트리스트 대조 + 코드 정규화 이중 검증으로 할루시네이션을 원천 차단했습니다.

### 2) 현장 원칙이 내장된 AI 평가 루브릭 생성

성취기준별 평가 요소·평가 방법·상/중/하 성취수준·피드백 방향을 한 번에 설계합니다. "'상'은 단순히 잘한다가 아니라 질적으로 구별되는 수행 특성", "'하'는 결핍 서술이 아닌 교사 지원 조건('도움을 받아 ~한다')으로 기술" 등 수석교사급 루브릭 작성 원칙을 프롬프트 수준에서 강제했습니다. 모든 셀은 클릭 즉시 인라인 수정되며, 수정 내용은 다음 단계에 그대로 반영됩니다.

### 3) 결재 문서 표준 양식의 백워드 수업 과정안 + A4 인쇄 최적화

확정된 루브릭으로부터 거꾸로 수업을 설계합니다. 교육과정 분석(핵심 아이디어·성취기준), 탐구 질문, 학습 목표, 수업자의 의도, 평가 계획표, 그리고 도입-전개-정리 교수·학습 과정안(자료 ▣ · 유의점 ※ · 평가 ☞ 기호 체계)까지 학교 현장 결재 문서 표준 양식 그대로 생성합니다. "모든 학습 활동은 루브릭의 평가 요소를 증명하는 장면일 것"이라는 백워드 원칙을 강제하며, [인쇄하기] 클릭 시 섹션별 페이지 분리·표 헤더 반복이 적용된 A4 결재 문서로 출력됩니다.

### 4) 수업별 코드·QR 기반 학생 평가 수합 (로그인 불필요)

교사가 [평가 시작]을 누르면 수업별 6자리 참여 코드와 QR이 발급됩니다. 학생은 설치·로그인 없이 QR을 찍고 이름만 입력하면 참여할 수 있으며, 루브릭에서 설정한 평가 방법이 그대로 문항 유형으로 자동 변환됩니다(서술·구술·관찰 → 서술형, 자기평가 → 상/중/하 선택형+까닭, 동료평가 → 동료 관찰형). 이때 교사용 전문 용어로 된 평가 요소를 AI가 초등학생 눈높이의 쉬운 질문으로 자동으로 다듬어 제시하고, 제출은 Firestore 실시간 동기화로 교사 화면에 즉시 쌓입니다.

### 5) AI 평가·피드백과 '교사 승인 게이트'

제출된 답안을 AI가 루브릭의 상/중/하 기준 문구와 대조하여 수준을 판정하고, 학생 이름을 부르며 시작하는 성장 중심 피드백 초안과 교사용 판정 근거를 생성합니다. 그러나 AI의 결과는 어디까지나 초안입니다. 교사가 수준과 피드백을 검토·수정한 뒤 [승인하고 전송]을 눌러야만 학생 화면에 전달되는 승인 게이트 구조로 교사의 평가권을 보장했습니다. 전송된 피드백은 다문화 학생을 위해 영어·중국어·베트남어·러시아어 등 8개 언어로 즉시 번역되며, 한국어 원문이 함께 표시되어 언어 학습을 돕습니다.

### 6) 학교생활기록부 기재요령 엄수 문구 생성

승인 완료된 학생별 평가 기록을 2026학년도 초등학교 생기부 기재요령을 엄수한 '교과학습발달상황' 문구로 변환합니다. 개조식/명사형 종결('~함.', '~하는 모습이 돋보임.'), 점수·등수 등 정량 데이터·수상 실적·사교육 요인 기재 금지, '하' 수준 학생도 관심·참여·성장 가능성 중심의 긍정 서술 등 공식 기재요령이 프롬프트에 규칙으로 내장되어 있습니다. 목표 글자 수 설정과 실시간 카운터, [NEIS 복사] 버튼으로 나이스 입력까지 한 번에 이어집니다.

### 7) 구글 로그인 & 다중 수업 관리

구글 계정으로 로그인하면 성취기준·루브릭·과정안·진행 단계가 1.5초 디바운스로 자동 저장되고, 교사 한 계정으로 여러 수업을 만들어 수업마다 진행 상황을 이어서 작업할 수 있습니다.

## 3. 강력한 안전 및 프라이버시 보호

- API 키 보안: Gemini API 키를 클라이언트에 두지 않고 Vercel Serverless Functions(`/api`) 환경 변수로 완벽 분리하여 브라우저 노출을 원천 차단했습니다.
- 할루시네이션 이중 방어: AI 응답을 성취기준 화이트리스트·코드 정규화로 검증하고, 긴 응답의 JSON 규격 위반·잘림을 스스로 복구하는 JSON 자동 수리기를 탑재했습니다.
- 모델 무중단 자동 탐색: 사용 가능한 Gemini 모델 목록을 실시간 조회하여 최신 모델을 자동 선택 — Google의 모델 세대 교체에도 코드 수정 없이 동작합니다.
- 학생 개인정보 최소 수집: 학생은 로그인 없이 이름(번호)만으로 참여하며, 입력창에 개인정보를 넣지 않도록 이용약관·개인정보처리방침에 명시했습니다. AI 피드백은 교사 승인 전까지 학생에게 노출되지 않습니다.

---

## 4. 기술 스택 (Tech Stack)

- Frontend: React 18 (Vite), Tailwind CSS (파스텔톤 에듀테크 디자인 시스템, `@media print` A4 최적화), Lucide React
- Backend / DB: Firebase (Google Auth, Firestore 실시간 동기화), Vercel Serverless Functions (AI 프록시 및 키 보안 계층)
- AI Integration: Google Gemini API (모델 자동 탐색, thinking 제어, 다단계 프롬프트 체이닝)
- Data: 2022 개정 교육과정 초등 핵심 성취기준 통합 데이터 611건 (학년군·과목·영역·핵심 아이디어 구조화)
- Version Control & Deployment: Git, GitHub, Vercel (푸시 자동 배포)

---

## 5. 기대 효과

성취기준 탐색부터 루브릭·과정안 작성, 평가지 배부·수합·채점, 생기부 문구 작성까지 수 시간씩 걸리던 평가 문서 작업을 몇 분 단위로 단축하여, 교사가 문서가 아닌 학생의 성장 자체에 집중할 시간을 돌려드립니다. 평가와 수업이 하나의 데이터 흐름으로 이어지는 백워드 구조는 '가르친 것과 평가하는 것이 다른' 불일치를 시스템 차원에서 제거하며, 학생은 제출 즉시 자신의 이름이 불리는 개별 피드백을 받고, 다문화 학생은 언어 장벽 없이 같은 피드백을 누립니다. 평가가 수업의 끝이 아니라 성장의 시작이 되는 교실을 만들어갈 것입니다.
